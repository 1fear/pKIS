Инструкция по файлам TakSklad

Для работы приложения после версии 1.1.7 используется папочная сборка (--onedir).
Обычно на компьютере склада будет папка TakSklad, а внутри нее файл TakSklad.exe
и служебная папка _internal.

Для работы приложения рядом с TakSklad.exe должны лежать:

1. TakSklad.exe
   Само приложение для склада.
   Запускать нужно именно этот файл или ярлык TakSklad на рабочем столе.

2. _internal
   Служебная папка собранного приложения.
   Ее нельзя удалять: внутри лежат библиотеки Python и зависимости.

3. credentials.json
   Ключ доступа к Google Sheets.
   Без этого файла приложение не сможет загрузить заказы и записать КИЗы.
   Этот файл нельзя отправлять посторонним.

4. TakSklad_data.json
   Общий файл данных приложения.
   Внутри хранятся настройки Telegram, очереди записи/печати/Telegram,
   настройки печати, справочник товаров, история импортов и, если перенесено,
   содержимое credentials.json.

При первом запуске собранная Windows-версия сама создаст ярлык TakSklad
на рабочем столе текущего пользователя. Складским сотрудникам лучше запускать
приложение через этот ярлык, чтобы не путаться в файлах папки.

Во время работы рядом с TakSklad.exe могут появляться служебные файлы и папки:

1. TakSklad.log
   Журнал ошибок.
   Если приложение показало ошибку, не смогло записать данные, напечатать сводку
   или сделать отчет, подробности будут записаны сюда.

2. scan_backups
   Папка аварийных копий сканов.
   Приложение сохраняет туда информацию по сканам на случай сбоя.
   Файлы создаются по датам, например:
   scan_backup_14.05.2026.jsonl

3. reports
   Папка Excel-отчетов.
   После кнопки "Завершить день" сюда сохраняется отчет за день.
   Каждый отчет создается отдельным файлом с датой и временем, например:
   scan_report_14.05.2026_183012.xlsx

4. Старые JSON-файлы
   pending_saves.json, pending_prints.json, pending_telegram.json,
   telegram_state.json, product_catalog.json, import_history.json,
   print_settings.json и telegram_settings.json больше не обязательны.
   При первом запуске TakSklad перенесет их содержимое в TakSklad_data.json.
   Старые файлы можно оставить рядом как резервную копию.

9. yandex_geocoder_key.txt
   Необязательный файл для замены встроенного ключа Яндекс Геокодера.
   Обычно он не нужен: ключ уже встроен в приложение.
   Если когда-нибудь ключ поменяется, можно положить новый ключ в этот файл,
   и приложение будет использовать его вместо встроенного.

При импорте Excel, если адрес пустой, но заполнены координаты,
приложение попробует само получить адрес через Яндекс Геокодер.

Обычно сотруднику склада нужно запускать только TakSklad.exe.
Остальные файлы нужны для надежности, отчетов и разбора ошибок.

Автообновление после перехода на папочную сборку скачивает ZIP-пакет новой версии,
обновляет TakSklad.exe и _internal, но сохраняет локальные рабочие данные:
TakSklad_data.json, credentials.json, логи, отчеты и scan_backups.

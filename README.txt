Инструкция по файлам pKIS

Для работы приложения рядом должны лежать:

1. pKIS.exe
   Само приложение для склада.

2. credentials.json
   Ключ доступа к Google Sheets.
   Без этого файла приложение не сможет загрузить заказы и записать КИЗы.
   Этот файл нельзя отправлять посторонним.

Во время работы рядом с pKIS.exe могут появляться служебные файлы и папки:

1. pKIS.log
   Журнал ошибок.
   Если приложение показало ошибку, не смогло записать данные, напечатать сводку
   или сделать отчет, подробности будут записаны сюда.

2. scan_backups
   Папка аварийных копий сканов.
   Приложение сохраняет туда информацию по сканам на случай сбоя.
   Файлы создаются по датам, например:
   scan_backup_14.05.2026.jsonl

3. reports
   Папка Excel-отчетов.
   После кнопки "Завершить день" сюда сохраняется отчет за день.
   Каждый отчет создается отдельным файлом с датой и временем, например:
   scan_report_14.05.2026_183012.xlsx

4. pending_prints.json
   Очередь сводных листов, которые нужно допечатать.
   Если КИЗы уже записались в Google Sheets, но печать сводки была отменена
   или принтер не сработал, приложение сохранит сводку здесь.
   При следующем запуске приложение предложит напечатать такие сводки.

Обычно сотруднику склада нужно запускать только pKIS.exe.
Остальные файлы нужны для надежности, отчетов и разбора ошибок.
